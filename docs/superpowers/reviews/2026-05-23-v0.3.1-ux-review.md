# ConfigForge v0.3.1 人机工程与易用性测试报告

**测试日期**: 2026-05-23
**测试版本**: v0.3.1 (AI SQL 步骤链编排)
**测试视角**: 最终用户 — 挑剔、专业、有创意
**测试环境**: macOS / Chromium 1440×900 + 375×812 (移动端)
**测试方法**: 全流程走查 + 边界场景 + 视觉审查 + 交互一致性检查

| 维度 | 评分 | 说明 |

## 二、核心流程走查

### 2.1 首页 → 向导入口

**发现的问题**：

| # | 严重程度 | 问题 | 建议 |
|---|---------|------|------|
| H-01 | 🟡 中等 | Hero 文案 "用自然语言描述需求 AI 自动生成配置" 缺少换行和视觉层次，一坨文字堆在一起 | 拆分为标题 + 副标题，用不同字号/颜色区分 |
| H-02 | 🟢 轻微 | 示例提示词是按钮样式，但点击后跳转到向导而非自动填入描述 | 点击示例应直接跳转向导并预填描述 |
| H-03 | 🟡 中等 | 配置列表中存在大量重复测试数据（"Test Config" × 10+），影响首页专业感 | 添加"清空测试数据"功能或过滤机制 |

### 2.2 向导 Step 1 — 场景信息

**发现的问题**：

| # | 严重程度 | 问题 | 建议 |
|---|---------|------|------|
| W-01 | 🔴 严重 | **所有 5 个步骤同时可见**，页面上同时出现 21 个可操作按钮、4 个"保存并继续"按钮。用户无法聚焦当前任务 | 只展示当前步骤，已完成步骤折叠，未到达步骤隐藏 |
| W-02 | 🟡 中等 | "版本号"字段默认值 "1.0" 对新用户无意义，且非必填项占据首屏空间 | 将版本号移至高级选项或折叠区 |
| W-03 | 🟢 轻微 | 场景名称为空时"保存并继续"按钮变为 disabled，但无即时提示 | 在输入框旁实时显示"请输入场景名称"提示 |

### 2.3 向导 Step 2 — 输入源

**发现的问题**：

| # | 严重程度 | 问题 | 建议 |
|---|---------|------|------|
| W-04 | 🟡 中等 | 上传文件后默认表名为 "result"，这个名称对 AI 编排和 SQL 生成有误导性（AI 会用 "result" 作为表名生成 SQL） | 默认使用文件名（去掉扩展名），如 "test" → "test"，"employee_salary" → "employee_salary" |
| W-05 | 🟡 中等 | 上传后列信息以标签形式展示，但无法编辑、删除或手动添加列 | 支持列的增删改，AI 编排依赖准确的列信息 |
| W-06 | 🟢 轻微 | 输入源类型选择器中 Excel 选项显示为 "📊Excel.xlsx / .xls"，格式混乱 | 统一为 "Excel (.xlsx, .xls)"，图标用 SVG 而非 emoji |

### 2.4 AI 面板

**发现的问题**：

| # | 严重程度 | 问题 | 建议 |
|---|---------|------|------|
| A-01 | 🟡 中等 | AI 面板占屏幕宽度 23.6%（340px），主内容区被压缩到 76.4%。在 1440px 屏幕上勉强可用，1280px 屏幕会非常局促 | 考虑可拖拽调整宽度，或使用抽屉式覆盖而非并排 |
| A-02 | 🟡 中等 | AI 面板切换是顶部导航栏的一个小开关，新用户很难发现 | 在 Step 2/3 右侧添加浮动 AI 助手按钮，或首次使用时自动展开 |
| A-03 | 🟢 轻微 | 输入框 placeholder 为空，没有提示用户可以输入什么 | 添加 placeholder 如 "描述你想要的数据处理..." |
| A-04 | 🟢 轻微 | 快捷操作按钮样式为 "› AI 编排步骤链"，前缀 "›" 符号含义不明 | 使用图标或更明确的视觉标识，如 "🔗 编排步骤链" |

### 2.5 编排流程（v0.3.1 核心功能）

**测试结果**：

| 步骤 | 操作 | 结果 | 评价 |
|------|------|------|------|
| 1 | 点击 "AI 编排步骤链" | AI 提示输入描述 | ✅ 提示文案清晰，含示例 |
| 2 | 输入 "按部门统计人数和平均工资" | AI 返回 1 步编排结果 | ✅ 正确触发编排 |
| 3 | 查看编排结果卡片 | 显示步骤名、SQL、输入/输出表 | ⚠️ 见下方问题 |
| 4 | 点击 "确认并填入向导" | SQL 填入 Step 3 | ✅ 正确填入 |
| 5 | 点击 "重新生成" | AI 提示重新描述 | ✅ 正确进入重新编排模式 |

**编排结果展示问题**：

| # | 严重程度 | 问题 | 建议 |
|---|---------|------|------|
| O-01 | 🔴 严重 | **编排结果卡片内 SQL 代码与步骤信息挤在一起**，字体仅 11px，在 AI 面板 340px 宽度内几乎不可读 | SQL 代码块使用等宽字体 + 语法高亮 + 可折叠展开 + 最小高度保证 |
| O-02 | 🟡 中等 | 编排结果只显示 "确认" 和 "重新生成" 两个选项，**无法编辑单步 SQL**、无法删除某步、无法调整步骤顺序 | 每步添加 "编辑" 按钮（点击展开 SQL 编辑器）和 "删除" 按钮 |
| O-03 | 🟡 中等 | "确认并填入向导" 是不可逆操作，确认后无法撤回 | 添加确认对话框，或在填入后 Step 3 提供 "撤销 AI 填入" 按钮 |
| O-04 | 🟢 轻微 | 步骤卡片中 "入: test → 出: department_stats" 信息对非技术用户不友好 | 改为 "数据来源: test → 输出为: department_stats"，或用图标+tooltip 解释 |
| O-05 | 🟢 轻微 | 编排结果中 explanation 文本与步骤卡片混在一起，视觉层次不清 | explanation 独立展示在步骤列表上方，用不同背景色区分 |

### 2.6 Step 3 — 确认处理器步骤

**发现的问题**：

| # | 严重程度 | 问题 | 建议 |
|---|---------|------|------|
| W-07 | 🟡 中等 | AI 填入后，Step 3 的 SQL textarea 中原有空步骤仍保留，用户看到 2 个步骤（1 空 + 1 AI 生成） | `onOrchestrateConfirm` 应替换而非追加处理器，当前 `store.setProcessors` 已实现替换，但需确认是否有残留 |

### 2.7 错误处理

**测试结果**：

| 场景 | 实际行为 | 评价 |
|------|---------|------|
| 未上传文件时点击编排 | 编排按钮不显示 | ✅ 正确隐藏 |
| 未上传文件时手动输入 "编排步骤链" | 提示 "没有检测到已上传的输入源" | ✅ 错误提示清晰 |
| AI 返回格式异常 | 提示 "AI 返回格式异常" + 原始响应 | ✅ 有助于排查 |
| 网络错误 | 提示 "AI 请求失败，请确认 AI 设置" | ✅ 区分了网络和 AI 问题 |

**缺失的错误处理**：

| # | 严重程度 | 问题 | 建议 |
|---|---------|------|------|
| E-01 | 🟡 中等 | AI 编排请求期间（等待 30 秒）**无任何加载状态指示**，用户不知道是否在处理 | 添加 loading 动画 + "AI 正在规划..." 提示 + 取消按钮 |
| E-02 | 🟢 轻微 | AI 编排超时无处理，用户可能无限等待 | 设置前端超时（如 60 秒），超时后提示重试 |

## 三、视觉与交互一致性

### 3.1 中英文混杂 🔴 严重

| 位置 | 当前文案 | 建议修改 |
|------|---------|---------|
| Step 3 输入/输出表 placeholder | "Please Input" | "请输入表名" |
| Settings 页面 max_tokens placeholder | "Please Input" | "请输入最大 token 数" |
| 步骤导航 | "Step 1", "Step 2"... | "第 1 步", "第 2 步"... 或纯中文标题 |
| AI 面板 | "AI" 标签 | 可保留，但需与整体风格一致 |

**影响**：中文用户看到 "Please Input" 会感到困惑，降低产品专业度。

### 3.2 Emoji 作为图标 🟡 中等

当前使用 emoji 作为按钮图标：🌙（暗色模式）、✏️（新建）、📖（指南）、▶（运行）、🔍（预览）、✨（AI 生成）。

**问题**：
- Emoji 在不同操作系统上渲染不一致
- Emoji 无法精确控制大小和颜色
- 与 Naive UI 组件库的专业风格不协调

**建议**：使用 [Lucide Icons](https://lucide.dev/) 或 [Heroicons](https://heroicons.com/) 等 SVG 图标库，与 Naive UI 的 `<NIcon>` 配合使用。

### 3.3 按钮样式不统一 🟡 中等

| 按钮类型 | 样式 | 问题 |
|---------|------|------|
| "保存并继续 ↓" | 自定义 btn-primary | 有箭头 ↓ 但其他按钮没有 |
| "+ SQL 步骤" | Naive UI 默认 | 前缀 + 号风格不一致 |
| "✨ AI 生成 SQL" | Naive UI 默认 | emoji 前缀 |
| "确认并填入向导" | Naive UI NButton primary | 与自定义 btn-primary 风格不同 |
| "重新生成" | Naive UI NButton default | 与其他按钮风格统一 |

**建议**：统一使用 Naive UI 的 `<NButton>` 组件，通过 `type` 属性区分主要/次要/危险操作。

### 3.4 色彩体系 🟢 轻微

编排结果卡片使用 `border-green-200 bg-green-50/40` 绿色系，而其他 AI 相关 UI 使用蓝色系。建议统一 AI 生成内容的视觉标识色。

## 五、可发现性与引导

### 5.1 编排功能的可发现性 🟡 中等

**当前路径**：进入向导 → 填写场景名 → 添加输入源 → 上传文件 → 打开 AI 面板 → 看到 "AI 编排步骤链"

**问题**：
1. AI 面板开关在导航栏中，不够醒目
2. 新用户不知道 AI 面板的存在
3. 编排功能需要先完成 Step 1 + Step 2 才能使用，但没有任何引导

**建议**：
- 首次进入向导时，自动展开 AI 面板并高亮快捷操作
- 在 Step 3 空白状态下显示 "让 AI 帮你规划步骤" 的引导卡片
- 在 Step 2 上传文件后，弹出 toast 提示 "文件已就绪，可以使用 AI 编排步骤链了"

### 5.2 缺少操作引导 🟡 中等

- 首次使用时无新手引导（onboarding tour）
- 各步骤之间缺少上下文说明（"为什么需要这一步？"）
- AI 面板的快捷操作没有 tooltip 解释各功能的区别

## 六、创意建议

### 6.1 编排结果交互增强

**当前**：编排结果是一个静态卡片，只能确认或重新生成。

**建议**：
1. **可拖拽排序**：用户可以拖拽调整步骤顺序
2. **步骤间连线**：可视化展示数据流（输入表 → 步骤1 → 步骤2 → 输出）
3. **单步预览**：点击某步可预览该步 SQL 的执行结果（dry run）
4. **渐进式确认**：用户可以只确认部分步骤，其余手动调整

### 6.2 上下文感知的 AI 建议

**当前**：AI 面板的快捷操作是静态列表。

**建议**：
1. **动态推荐**：根据当前步骤和已有数据智能推荐操作（如 Step 2 上传后推荐 "AI 分析列"，Step 3 空白时推荐 "AI 编排步骤链"）
2. **上下文续写**：AI 聊天中可以引用之前的编排结果，如 "在上次编排的基础上，增加一个数据清洗步骤"
3. **历史记录**：保存用户的编排历史，支持 "使用上次编排"

### 6.3 协作式编排

**建议**：
1. 用户可以先手动创建 1-2 个步骤，然后让 AI 在此基础上继续编排
2. AI 编排时考虑已有的处理器步骤，避免冲突
3. 支持 "AI 补全" — 用户写了一半的 SQL，AI 帮助补全

## 九、下一步行动建议

### 第一优先级（影响核心体验）

1. **步骤折叠/展开机制**：只展示当前步骤，这是最大的 UX 问题
2. **AI 加载状态**：编排请求期间显示 loading + 进度提示
3. **全站中文化**：消除所有 "Please Input" 等英文残留

### 第二优先级（提升专业度）

4. **编排结果可编辑**：支持单步 SQL 编辑和步骤删除
5. **SVG 图标替换 emoji**：统一视觉风格
6. **默认表名优化**：使用文件名替代 "result"

### 第三优先级（锦上添花）

7. **移动端适配**：手风琴步骤 + 底部 AI 抽屉
8. **新手引导**：首次使用 onboarding tour
9. **上下文感知推荐**：智能推荐快捷操作
