## 三、需要调整的项

### 3.1 已修复（从方案中移除）

| # | 原因 | 证据 |
|---|------|------|
| M-2 | `configs.py` 的 `_load_index/_save_index` 已使用 `fcntl.flock` | 代码第63-76行 |

### 3.2 优先级调整

| # | 原优先级 | 建议优先级 | 原因 |
|---|---------|-----------|------|
| B-5 | BLOCKER | HIGH | 本地开发工具，认证非 P0 |
| H-2 | HIGH | BLOCKER | 内存溢出是生产风险 |
| H-7 | HIGH | BLOCKER | SQL 预览可注入 DROP TABLE |
| H-4 | HIGH | MEDIUM | TypeScript `any` 类型是技术债务，不影响功能 |
| H-5 | HIGH | MEDIUM | 后端分页已就绪，前端补 UI 即可 |
| M-13 | MEDIUM | HIGH | Pipeline 超时是稳定性关键 |

### 3.3 方案补充建议

| # | 补充内容 |
|---|---------|
| B-1 | 建议在 SELECT-only 方案上加 `--allow-write` CLI 参数，给高级用户留有后路 |
| H-7 | 建议将 `PRAGMA query_only` 方案改为：数据加载→设置只读→执行用户 SQL。当前顺序是先执行 SQL 后加载数据，存在窗口 |
| M-8 | `HomeView.vue` 搜索定时器需确认 `onUnmounted` 是否缺失（搜索结果页切换时定时器泄露） |
