## 第三轮新发现的问题（4 个，均为中/轻微级别）

### 🟠 新问题 E：`ChatMessage` 类型在两处定义，会导致 TypeScript 冲突

**涉及文件：** Task 3 Step 1, Task 4 Step 2

**现状：**
- `ChatMessage` 当前定义在 [AiChatPanel.vue:67](file:///Users/lixinyuan/code/CCTEST/configforge-web/src/components/wizard/AiChatPanel.vue#L67)
- `ConfigWizardView.vue` 从 AiChatPanel 导入：`import type { ChatMessage } from '../components/wizard/AiChatPanel.vue'`
- `wizard.ts` 中**没有** `ChatMessage` 定义

**问题：** Task 3 Step 1 在 `wizard.ts` 中新增了带 `orchestration?` 字段的 `ChatMessage`，但没有说明如何处理 AiChatPanel.vue 中的原有定义。如果两处都定义 `ChatMessage`，会导致：
1. `ConfigWizardView.vue` 从 `AiChatPanel.vue` 导入的 `ChatMessage` 没有 `orchestration` 字段 → TypeScript 报错
2. 从 `wizard.ts` 导入的 `ChatMessage` 有 `orchestration` 字段，但 `AiChatPanel.vue` 不识别 → 类型不兼容

**修复方案：** 在 `wizard.ts` 中定义完整的 `ChatMessage`（含 `orchestration?`），然后：
1. `AiChatPanel.vue` 删除本地 `ChatMessage` 定义，改为从 `wizard.ts` 导入
2. `ConfigWizardView.vue` 的导入路径从 `AiChatPanel.vue` 改为 `wizard.ts`

### 🟠 新问题 F：Task 5 的 quickAction handler 集成代码片段有歧义

**涉及文件：** Task 5 Step 2

**问题：** 计划中的代码片段：

```typescript
if (action === 'orchestrate' || text.includes('编排') || text.includes('步骤链')) {
  await doOrchestrate(text || action)
  return
}
```

这段代码混合了两个不同处理器的变量：
- `action` 是 `onAiQuickAction(action: string)` 的参数，值为 `'AI 编排步骤链'`
- `text` 是 `onAiSend(text: string)` 的参数，值为用户输入的文本

条件 `action === 'orchestrate'` 永远不会匹配，因为实际快捷操作字符串是 `'AI 编排步骤链'`。而 `text.includes('编排')` 属于 `onAiSend` 的逻辑，不应出现在 `onAiQuickAction` 中。

**修复方案：** 明确拆分为两个 handler 的集成：

```typescript
// 在 onAiQuickAction 中：
if (action === 'AI 编排步骤链') {
  onOrchestrateAction()
  return
}

// 在 onAiSend 中（可选，用于用户直接输入编排相关关键词）：
if (text.includes('编排') || text.includes('步骤链')) {
  await doOrchestrate(text)
  return
}
```

### 🟡 新问题 H：Task 4 Step 2 的 import 语句有误

**涉及文件：** Task 4 Step 2

**问题：** 计划写的是：

> Import `OrchestrationResult` and `OrchestrationResult as OrchestrationResultType` from types.

这有两个问题：
1. 同一文件不能有两个同名 import（`OrchestrationResult` 和 `OrchestrationResult as OrchestrationResultType`）
2. 组件名 `OrchestrationResult` 和类型名 `OrchestrationResult` 冲突——Vue SFC 中组件名和类型名相同时，需要用不同名字区分

**修复方案：**

```typescript
import OrchestrationResultCard from './OrchestrationResult.vue'
import type { OrchestrationResult } from '../../types/wizard'
```

在模板中使用 `<OrchestrationResultCard>`，在 props/emits 类型中使用 `OrchestrationResult`。
