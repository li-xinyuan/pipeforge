## 逐项修复详情

### 修复 1: WizardStepCard locked 状态 pointer-events

**文件:** `configforge-web/src/components/wizard/WizardStepCard.vue:58-61`

**问题:** `.wizard-step-card--locked { pointer-events: none; }` 阻止了 locked 状态下所有子元素的点击事件，导致 "+ SQL 步骤"、"保存并继续"、"保存配置" 等按钮均无法点击。

**修复:** `pointer-events: none` 仅应用于 body 和 footer：

```css
.wizard-step-card--locked {
  opacity: 0.55;
}
.wizard-step-card--locked .wizard-step-card__body,
.wizard-step-card--locked .wizard-step-card__footer {
  pointer-events: none;
}
```

### 修复 2: 向后兼容迁移条件

**文件:** `configforge/api/configs.py:27`

**问题:** `_migrate_state_dict()` 的条件 `"processors" not in state_dict` 在 `state_dict` 包含 `processors: []`（空数组，来自新版前端的默认值）时不满足，导致迁移失效。

**修复:** 改为 `not state_dict.get("processors")`，空数组也为 falsy：

```python
# 旧
if "processor" in state_dict and "processors" not in state_dict:

# 新
if "processor" in state_dict and not state_dict.get("processors"):
```

### 修复 3: 首页缺少"编辑"菜单项

**文件:** `configforge-web/src/views/HomeView.vue:187-197`

**问题:** 配置卡片的下拉菜单只有"下载 YAML"和"删除"，缺少"编辑"选项。用户无法从首页直接进入编辑已有配置。

**修复:** 添加"编辑"选项，复用已有 `onLoadConfig` 方法：

```typescript
function getMenuOptions(cfg: SavedConfig) {
  return [
    { label: '编辑', key: 'edit' },
    { label: '下载 YAML', key: 'download' },
    { label: '删除', key: 'delete' },
  ]
}

function onMenuSelect(key: string, cfg: SavedConfig) {
  if (key === 'edit') onLoadConfig(cfg.id)
  else if (key === 'download') onDownloadYaml(cfg.id)
  else if (key === 'delete') promptDelete(cfg)
}
```

### 修复 4: sourceTable 下拉包含输入源表名

**文件:** `configforge-web/src/components/step3/OutputConfigTab.vue:199-207`

**问题:** `sourceTableOptions` 仅收集 processor 的 `outputTables`，但 processor 未填写输出表名时下拉为空。

**修复:** 同时包含输入源表名作为备选，去重显示：

```typescript
const sourceTableOptions = computed(() => {
  const tables: Array<{ label: string; value: string }> = []
  const seen = new Set<string>()
  for (const proc of store.processors) {
    for (const t of proc.outputTables) {
      if (t && !seen.has(t)) { seen.add(t); tables.push({ label: t + ' (处理输出)', value: t }) }
    }
  }
  for (const inp of store.inputs) {
    if (inp.table && !seen.has(inp.table)) { seen.add(inp.table); tables.push({ label: inp.table + ' (输入源)', value: inp.table }) }
  }
  return tables
})
```

### 修复 5: AI 面板更新当前展开的处理器

**文件:** `configforge-web/src/views/ConfigWizardView.vue:364-369`, `configforge-web/src/components/step3/SqlEditorTab.vue:241`

**问题:** AI 生成 SQL 后始终更新 `store.processors[0]`，多步骤场景下无法指定目标步骤。

**修复:**
1. SqlEditorTab 暴露 `expandedIndex` 供父组件读取
2. ConfigWizardView 使用当前展开的处理器索引，fallback 为最后一个处理器：

```typescript
// SqlEditorTab.vue — 暴露 expandedIndex
defineExpose({ checkTableRenames, expandedIndex })

// ConfigWizardView.vue — 使用展开索引
const targetIndex = sqlEditorRef.value?.expandedIndex ?? store.processors.length - 1
store.updateProcessor(targetIndex, proc)
```

### 修复 6: ProcessorCard data-testid 属性

**文件:** `configforge-web/src/components/step3/ProcessorCard.vue`

**问题:** Playwright 难以定位 Naive UI 的 `NInput` 组件（渲染为 div 包裹的 input）。

**修复:** 关键输入框添加 `data-testid` 属性：

| 元素 | data-testid |
|------|------------|
| 步骤名称 NInput | `processor-name-{index}` |
| 输出表名 NInput | `processor-output-{index}-{ti}` |
| SQL NInput | `processor-sql-{index}` |
