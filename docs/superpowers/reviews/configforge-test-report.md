## 🐛 问题清单 (按严重程度排序)

### 🔴 P0 - 严重问题 (必须修复)

#### 1. 引导步骤数不一致
- **位置**: 首页 `HomeView.vue`
- **问题**: 首页描述写的是 **"4 步向导"**，但实际是 **5 步**（场景信息 → 数据源配置 → 数据转换 → 输出定义 → 预览导出）
- **影响**: 用户产生认知混乱，信任度降低
- **源码**: `HomeView.vue:12` — `<strong>4 步向导</strong>`
- **建议**: 改为 "5 步向导"

#### 2. 品牌名称混用
- **位置**: 首页 `HomeView.vue`
- **问题**: 页面标题是 **"ConfigForge"**，但副标题和描述中使用了 **"PIPEFORGE"** 和 **"PipeForge"**
- **影响**: 品牌认知混乱，用户不知道产品到底叫什么
- **源码**: `HomeView.vue:4` — `ConfigForge`；`HomeView.vue:5` — `PIPEFORGE 配置创建向导`；`HomeView.vue:12` — `PipeForge 可执行`
- **建议**: 统一使用一个品牌名

#### 3. 无效路由白屏（无 404 处理）
- **位置**: `router/index.ts`
- **问题**: 
  - 访问 `/step/99` → 显示空白页面
  - 访问 `/nonexistent` → 显示空白页面
  - 没有定义 catch-all 路由或 404 页面
- **影响**: 用户通过书签/直接输入 URL 进入无效路径时看到白屏，以为是 bug
- **源码**: `router/index.ts` — 没有 `{ path: '/:pathMatch(.*)*', component: NotFoundView }` 路由
- **建议**: 添加 404 页面和 catch-all 路由

#### 4. 下拉菜单项不可交互（无障碍/可用性问题）
- **位置**: 首页配置列表 "···" 菜单
- **问题**: 点击 "···" 后显示的"下载 YAML / 执行 / 删除"在无障碍树中仅渲染为纯文本（`text: 下载 YAML 执行 删除`），**没有 button 角色**，键盘用户和屏幕阅读器无法操作
- **影响**: 
  - 键盘导航用户无法选择菜单项
  - 屏幕阅读器无法识别可操作选项
  - 浏览器自动化测试无法模拟点击
- **源码**: `HomeView.vue:50` — `NDropdown` 组件
- **建议**: 使用 Naive UI 的 `NDropdown` 时确保菜单项有正确的 role 属性，或改用 `<NButton>` 渲染选项

#### 5. 取消按钮行为异常
- **位置**: Step 2 取消按钮
- **问题**: 点击"取消"按钮后浏览器会话被终止（SIGTERM/SIGKILL），行为不明确——应该弹确认框还是直接返回首页？
- **影响**: 用户可能意外丢失已填写的数据
- **建议**: 添加确认对话框，明确提示"未保存的修改将丢失"

### 🟠 P1 - 重要问题 (建议修复)

#### 6. 表单输入缺少关联的 `<label>`
- **位置**: Step 1 场景信息表单 `SceneInfoForm.vue`
- **问题**: "场景名称"、"版本"、"场景描述" 等标签是独立的 `<text>` 节点，与 `<textbox>` 没有通过 `for`/`id` 关联
- **影响**: 屏幕阅读器无法正确播报标签，点击标签不会聚焦输入框
- **源码**: `Step1SceneView.vue` 快照显示 `text: 场景名称` 和 `textbox "输入场景名称"` 是两个独立节点
- **建议**: 使用 `<label for="xxx">` 或 `aria-labelledby` 关联

#### 7. 版本输入框预填 "1.0" 但无占位提示
- **位置**: Step 1
- **问题**: 版本字段 textbox 直接预填 "1.0" 作为值，而非 placeholder。用户可能误以为这是不可编辑的
- **建议**: 要么用 placeholder 提示，要么加帮助文本说明

#### 8. 首页配置名称大量重复
- **位置**: 首页
- **问题**: 6 个配置中有 4 个名为 "Test Config"，2 个分别叫 "Test" 和 "测试数据人员清洗筛选"
- **影响**: 用户无法区分不同配置，建议创建时强制唯一名称或自动添加序号
- **建议**: 添加名称唯一性校验或自动编号

#### 9. "管理 →" 链接为纯文本不可点击
- **位置**: 首页 AI 状态栏 `AiStatusBanner.vue`
- **问题**: "AI 已启用 · custom/qwen3.5-plus 管理 →" 整体通过 `<div @click>` 跳转，但 "管理 →" 在无障碍树中不是独立的 link/button，语义不清
- **源码**: `AiStatusBanner.vue:23-24` — 整个 div 绑 click 事件
- **建议**: 将 "管理 →" 改为真正的 `<NButton text>` 或 `<a>` 标签

#### 10. API Key 明文存储在 localStorage
- **位置**: `SettingsPage.vue` + Pinia persistedstate
- **问题**: API Key 通过 `pinia-plugin-persistedstate` 存储在 `localStorage` 中，任何页面脚本都可读取
- **影响**: XSS 攻击可轻易窃取 API Key
- **建议**: 考虑使用 httpOnly cookie 或加密存储
