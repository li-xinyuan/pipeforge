## 五、发现的 Bug

### Bug 1：`aiQuickActions` 条件逻辑语义错误 🔴 严重

**涉及文件**: `configforge-web/src/views/ConfigWizardView.vue` 第 274 行

**当前代码**：

```typescript
const aiQuickActions = computed(() => {
  const actions = ['生成场景描述', 'AI 分析列', 'AI 生成 SQL', 'AI 自动映射']
  if (store.inputs.length > 0 && store.processors.length > 0) {
    actions.unshift('AI 编排步骤链')
  }
  return actions
})
```

**问题**：

条件 `store.processors.length > 0` 语义上不正确。编排功能的目的是帮用户**创建**处理器步骤链，不应要求用户已有处理器。当前之所以"碰巧"能工作，是因为 store 初始化时 processors 默认包含一个空步骤，使得 `processors.length` 始终 > 0。但如果用户删除了所有处理器步骤（未来可能支持），或者 store 初始化逻辑改变，此条件就会失效。

**修改建议**：

```typescript
const aiQuickActions = computed(() => {
  const actions = ['生成场景描述', 'AI 分析列', 'AI 生成 SQL', 'AI 自动映射']
  if (store.inputs.length > 0) {
    actions.unshift('AI 编排步骤链')
  }
  return actions
})
```

移除 `store.processors.length > 0` 条件，只保留 `store.inputs.length > 0`。

### Bug 2：编排模式状态未保持，用户描述无法路由到 `doOrchestrate` 🔴 严重

**涉及文件**: `configforge-web/src/views/ConfigWizardView.vue` 第 337-341 行

**当前代码**：

```typescript
async function onAiSend(text: string) {
  aiMessages.value.push({ role: 'user', content: text })

  if (text.includes('编排') || text.includes('步骤链')) {
    await doOrchestrate(text)
    return
  }
  // ... 走普通聊天流程
}
```

**问题**：

用户点击 "AI 编排步骤链" 后，AI 提示用户输入自然语言描述（如"统计各部门的出勤天数和平均薪资"）。但用户输入后，`onAiSend` 仅通过关键词匹配（"编排"/"步骤链"）路由到 `doOrchestrate`。用户的自然语言描述通常不包含这些关键词，导致走了普通聊天流程，而非编排流程。

**E2E 测试验证**：

| 用户输入 | 是否触发 doOrchestrate | 说明 |
|---------|----------------------|------|
| "统计各部门的出勤天数和平均薪资" | ❌ | 不含关键词，走聊天流程 |
| "帮我编排步骤链，统计各部门出勤" | ✅ | 包含"编排"和"步骤链" |
| "请编排一个多步处理" | ✅ | 包含"编排" |

**修改建议**：

添加 `orchestrateMode` 状态标志，在用户点击 "AI 编排步骤链" 时设为 `true`，`onAiSend` 检测到该标志后自动路由到 `doOrchestrate` 并重置标志：

```typescript
// 在 aiMessages 定义后添加
const orchestrateMode = ref(false)

// 修改 onAiSend
async function onAiSend(text: string) {
  aiMessages.value.push({ role: 'user', content: text })

  if (orchestrateMode.value || text.includes('编排') || text.includes('步骤链')) {
    orchestrateMode.value = false
    await doOrchestrate(text)
    return
  }
  // ... 后续聊天逻辑不变
}

// 修改 onOrchestrateAction
function onOrchestrateAction() {
  if (store.inputs.length === 0) {
    aiMessages.value.push({ role: 'ai', content: '请先在 Step 2 添加输入源并上传文件。' })
    return
  }
  orchestrateMode.value = true  // ← 新增
  aiMessages.value.push({ role: 'ai', content: '请用中文描述你想要的最终报表...' })
}

// 修改 onOrchestrateRegenerate
function onOrchestrateRegenerate() {
  orchestrateMode.value = true  // ← 新增
  aiMessages.value.push({ role: 'ai', content: '请重新描述你的需求，我会重新规划处理链。' })
}
```

### Bug 3：`doOrchestrate` 错误处理不精确 🟡 中等

**涉及文件**: `configforge-web/src/views/ConfigWizardView.vue` 第 497-514 行

**当前代码**：

```typescript
async function doOrchestrate(naturalLanguage: string) {
  const context = {
    inputs: store.inputs.map(inp => {
      const meta = store.uploadedFiles[inp.fileId]
      return { table: inp.table, columns: meta?.columns || [] }
    }),
    outputColumns: (store.output?.config?.columns || []).map(c => c.target),
    naturalLanguage,
  }
  const result = await askOrchestrate(context)
  if (result && result.steps.length > 0) {
    aiMessages.value.push({ role: 'ai', content: '', orchestration: result })
  } else if (result?.parse_error) {
    aiMessages.value.push({ role: 'ai', content: 'AI 返回格式异常，请重试。原始响应：' + (result.raw || '').slice(0, 500) })
  } else {
    aiMessages.value.push({ role: 'ai', content: '无法规划处理链，请确保已上传文件并加载了列信息，然后重试。' })
  }
}
```

**问题 1**：`store.inputs.map(...)` 会包含没有 fileId 或 table 为空的输入源，导致发送给 AI 的上下文包含空表名，影响 AI 生成质量。

**问题 2**：`result` 为 `null`（请求失败）和 `result` 存在但 `steps` 为空（AI 无法规划）使用相同的错误消息，用户无法区分是网络问题还是 AI 能力问题。

**修改建议**：

```typescript
async function doOrchestrate(naturalLanguage: string) {
  const inputsContext = store.inputs
    .filter(inp => inp.fileId)
    .map(inp => {
      const meta = store.uploadedFiles[inp.fileId]
      return { table: inp.table, columns: meta?.columns || [] }
    })
    .filter(inp => inp.table)

  if (inputsContext.length === 0) {
    aiMessages.value.push({ role: 'ai', content: '没有检测到已上传的输入源，请先在步骤 2 上传文件并确认列信息。' })
    return
  }

  const context = {
    inputs: inputsContext,
    outputColumns: (store.output?.config?.columns || []).map(c => c.target),
    naturalLanguage,
  }
  const result = await askOrchestrate(context)
  if (result && result.steps.length > 0) {
    aiMessages.value.push({ role: 'ai', content: '', orchestration: result })
  } else if (result?.parse_error) {
    aiMessages.value.push({ role: 'ai', content: 'AI 返回格式异常，请重试。原始响应：' + (result.raw || '').slice(0, 500) })
  } else if (result) {
    aiMessages.value.push({ role: 'ai', content: result.explanation || 'AI 无法根据当前信息规划处理链，请补充更多细节后重试。' })
  } else {
    aiMessages.value.push({ role: 'ai', content: 'AI 请求失败，请确认 AI 设置正确且后端服务正在运行，然后重试。' })
  }
}
```

主要变更：
1. 过滤掉无 fileId 或 table 为空的输入源
2. 前置校验：无有效输入源时直接提示，不发起 API 请求
3. 区分三种失败情况：parse_error / result 存在但无步骤 / result 为 null
