## 二、问题清单

### 🟡 问题 1: F1 描述有误 — WizardStepCard 缺少 `step` prop

**计划原文**: "根据已有 `step` prop（值 1-5）绑定不同 CSS 类"

**实际情况**: `WizardStepCard.vue` 的 props 为 `{ title, description, icon, status, badge }`，**没有 `step` prop**。

**建议**: 需新增 `step` prop，或在 `ConfigWizardView.vue` 中传入 `iconColor` prop。推荐后者——步骤号是父组件的职责，卡片组件不应感知步骤语义。具体方案：

```vue
<!-- WizardStepCard.vue 新增 prop -->
defineProps<{
  title: string
  description: string
  icon: string
  status: 'completed' | 'active' | 'locked'
  badge?: string
  iconColor?: string   // 新增：图标区背景色
}>()
```

```vue
<!-- 使用时 -->
<WizardStepCard icon-color="var(--step-1-bg)" ... />
```

### 🟡 问题 2: F2 触发条件需细化

**计划原文**: Step2 条件是"有文件已上传且有列信息"

**实际情况**: `store.uploadedFiles` 中的 `columns` 是异步获取的，可能在文件预览 API 返回后才填充。

**建议**: 触发条件应更精确：

```typescript
// Step2 条件
store.inputs.some(inp => inp.fileId && store.uploadedFiles[inp.fileId]?.columns?.length)

// Step3 条件
store.inputs.length > 0 && !store.processor.sql.trim()

// Step4 条件
store.processor.sql.trim() && !(store.output?.config as any)?.columns?.length

// Step5 条件
yamlPreviewRef?.yamlText
```

---

### 🟡 问题 3: E1 多处理器链式执行需补充错误处理考虑

**计划原文**: 去掉 `[:1]` 切片限制

**实际情况**: 当前 `engine.py` 的 `try/except` 结构中，如果第二个处理器失败，第一个处理器的 stats 已 append 到 `context.result.processors`，但异常会中断后续 output 执行。

**建议**: 
- 确认期望行为：第二个处理器失败时，是否仍执行 output？
- 推荐保持当前行为（任一处理器失败则整个流水线失败），这是最安全的策略
- 在验收标准中增加："第二个处理器失败时，第一个处理器的 stats 仍可在 context.result 中查看"

---

### 🟡 问题 4: E2 执行日志持久化缺少清理策略

**计划原文**: `pipeforge_{timestamp}.log`

**实际情况**: 频繁执行会积累大量日志文件，计划中没有清理策略。

**建议**:
- 添加日志文件数量上限（如保留最近 50 个）
- 或添加过期清理（如 7 天后自动删除，与 `files.py` 的 24h 清理逻辑一致）
- 在 `files.py` 的 `cleanup_old_files()` 旁边增加 `cleanup_old_logs()`

---

### 🟡 问题 5: 遗漏 N1 — AI 列分析缺少 sampleRows 传递

**现状**: `FIX_VERIFICATION_REPORT` 中标记为 🟡 中等优先级，但 v0.2 计划未包含。

**建议**: 在 F2（AI 内联建议 Step2-5）实现中顺带修复，改动量极小（约 3 行）：

```typescript
// ConfigWizardView.vue onAiQuickAction('AI 分析列')
const result = await askSuggestion('columns', {
  inputs: filesWithColumns.map(inp => ({
    name: inp.table,
    table: inp.table,
    columns: store.uploadedFiles[inp.fileId]!.columns,
    sampleRows: store.uploadedFiles[inp.fileId]?.sampleRows || [],  // 新增
  })),
})
```

## 四、结论

**计划整体可行**，建议做以下 3 处调整后再启动开发：

1. **修正 F1 描述** — WizardStepCard 没有 `step` prop，需新增或改用 `iconColor` prop
2. **补充 N1 修复** — 在 F2 实现中顺带传递 `sampleRows`，改动极小但收益显著
3. **补充 E2 日志清理** — 添加日志文件过期清理策略，与现有 `cleanup_old_files()` 模式一致

其余内容（E1 去掉切片、F2 触发条件、验收标准）均合理，可直接执行。
