# ConfigForge AI 引导式配置 — 交互设计文档

> 版本: v1.0
> 日期: 2026-06-07
> 状态: 设计阶段，待确认后进入开发计划

---

## 一、设计目标

将 ConfigForge 从"手动填写表单 + AI 可选辅助"改造为"AI 手把手引导 + 用户审核确认"的协同配置模式。

**核心原则：**

1. **AI 主导引导，用户审核确认** — AI 主动提问、主动填写、主动推进流程，用户只需确认或微调
2. **对话与界面联动** — 用户在对话中的选择直接反映到表单，操作表单也会被 AI 感知并响应
3. **渐进式披露** — 步骤按序解锁，每步由 AI 引导完成后再进入下一步
4. **多模态输入** — 支持文字输入、选项按钮点击、语音输入三种方式，效果一致
5. **首页简洁** — 只有一个输入框，AI 面板在进入向导后才出现

**关键规则：**

- 首页：只有输入框，**不显示 AI 面板**
- 进入向导后：AI 面板从右下角出现，可拖拽
- 步骤按序解锁：当前步骤完成才解锁下一步
- 已完成的步骤可回看，但 AI 引导焦点始终在当前步骤

## 四、5 步向导 + AI 协同

### 4.1 进度指示器

```
[✓ 场景信息] → [⟳ 输入源] → [ 处理步骤] → [ 输出配置] → [ 导出执行]
   已完成       当前步骤      待解锁        待解锁        待解锁
```

- 已完成：绿色边框 + ✓ 图标
- 当前步骤：黄色边框 + 脉冲动画
- 待解锁：灰色，不可点击

### 4.2 步骤 1 — 场景信息

**AI 行为：**
- 进入时 AI 面板自动弹出第一条消息
- AI 从用户首页输入中提取场景名称，自动填入表单
- AI 生成场景描述，自动填入
- AI 提示用户确认或修改

**AI 面板消息示例：**
```
🤖 好的！我来帮你创建这个订单统计流水线。
   先看步骤 1——我已经帮你填好了场景名称，
   你可以直接修改。确认后我们进入步骤 2。

   [✅ 确认，下一步]
```

**表单状态：**
- 场景名称：已填写，绿色边框（AI 填充标记）
- 场景描述：已填写
- 版本号：默认 "1.0"

### 4.3 步骤 2 — 输入源

**AI 行为：**
- AI 面板询问数据来源类型
- 同时，页面上的三个类型卡片可见
- 用户可以通过三种方式选择：
  1. 直接点击页面上的类型卡片
  2. 点击 AI 面板中的选项按钮
  3. 在 AI 面板输入框中打字（如 "Excel"）

**三种输入方式联动：**
```
用户点击卡片 "📊 Excel"
  → 卡片高亮，显示上传区域
  → AI 面板自动追加用户消息 "📊 Excel"
  → AI 面板回复 "好的，Excel 文件。请上传文件..."

用户在 AI 面板输入 "CSV"
  → AI 识别意图，面板显示选项确认
  → 页面上 CSV 卡片自动高亮
  → 上传区域切换到 CSV 模式

用户在 AI 面板点击按钮 "📊 Excel"
  → 同点击卡片的效果
```

**AI 面板消息示例：**
```
🤖 现在我们配置输入源。
   你的数据在哪种文件或系统里？
   可以直接点上方卡片，也可以在下面告诉我～

   [📊 Excel] [🗄 CSV] [🔌 数据库]

👤 📊 Excel

🤖 好的，Excel 文件。请上传文件，
   然后我们进入步骤 3 生成处理代码。

   [📤 我上传好了]
```

**表单状态：**
- 选择前：三个卡片等大，虚线边框
- 选择后：选中卡片实线绿色边框 + 浅绿背景，其余变暗
- 上传区域出现（虚线框 + 拖拽提示）

### 4.4 步骤 3 — 处理步骤

**AI 行为：**
- AI 根据场景描述 + 输入源信息，自动生成 SQL 或 Python 代码
- 代码显示在可编辑的代码块中
- AI 解释代码逻辑，标注假设（如关联字段名）
- 用户可编辑代码，或通过对话让 AI 修改

**AI 面板消息示例：**
```
🤖 我已经生成了 SQL 代码：
   用 JOIN 关联两个表，按城市分组统计订单数和金额。
   我假设关联字段是 user_id，不对的话在代码块里直接改～

   [✅ 确认，下一步] [✏ 我要修改]

👤 把统计改成按产品类型分组

🤖 好的，SQL 已更新为 GROUP BY product_type。
   Step 4 的列映射我也同步更新了。

   [✅ 确认]
```

**跨步骤联动（关键设计）：**
- 用户修改 SQL 的输出列 → AI 检测到变化 → 主动提示是否需要同步更新 Step 4 的列映射
- 气泡样式：⚠ 警告色（黄色边框），需要用户确认

### 4.5 步骤 4 — 输出配置

**AI 行为：**
- AI 根据场景描述自动选择输出类型（Excel/CSV/数据库）
- AI 自动生成文件名模板
- AI 自动完成列映射（从 SQL SELECT 字段推断）
- 支持切换输出类型

**AI 面板消息示例：**
```
🤖 输出配置好了：Excel 格式，
   列映射已自动完成：city, order_count, total_amount
   我们进入最后一步吧！

   [✅ 确认] [🔄 换成 CSV]
```

**表单状态：**
- 输出格式：下拉选择（Excel / CSV / 数据库）
- 文件名：已填写，带模板变量（{{date}}）
- 列映射：表格显示，AI 已完成，可手动编辑
- 数据检查点：可选，AI 建议添加

### 4.6 步骤 5 — 导出执行

**AI 行为：**
- 确认全部配置完成
- 提供下载 YAML / 执行流水线两个操作

**AI 面板消息示例：**
```
🤖 🎉 全部配置完成！5 个步骤都就绪了。
   你可以下载 YAML 文件，或直接执行流水线。

   [🎉 太棒了]
```

## 五、AI 面板设计

### 5.1 外观

```
┌──────────────────────────────┐
│ 🤖 Forge · AI 助手      [−] │  ← 拖拽手柄（可拖动面板）
├──────────────────────────────┤
│                              │
│  🤖 好的！我来帮你...         │  ← AI 消息（深色气泡）
│                              │
│              👤 Excel 文件    │  ← 用户消息（绿色气泡）
│                              │
│  🤖 收到。请上传文件...       │
│     [📤 我上传好了]           │  ← 快捷操作按钮
│                              │
├──────────────────────────────┤
│ 🎤 [________________] [发送] │  ← 输入区
└──────────────────────────────┘
```

### 5.2 特性

| 特性 | 说明 |
|------|------|
| 跟随定位 | 默认跟随当前步骤卡片右侧，可拖拽；遮盖"下一步"按钮时自动避让 |
| 可最小化 | 点击 `−` 缩成 🤖 悬浮图标，双击恢复 |
| 自适应高度 | 最大 500px，消息多时内部滚动 |
| 快捷按钮 | AI 消息可附带选项按钮，点击即回复 |
| 语音输入 | 所有输入框旁 🎤 按钮（当前版本为占位符，下一版本实现） |
| 上下文感知 | 面板始终知道当前步骤，消息与步骤关联 |
| 历史持久化 | 切换步骤和刷新页面后对话历史保留 |

### 5.3 消息类型

| 类型 | 样式 | 用途 |
|------|------|------|
| 普通消息 | 深色气泡 | AI 的说明、提问 |
| 建议气泡 | 青色边框 | AI 的建议、提示 |
| 警告气泡 | 黄色边框 | 检测到关联影响，需用户确认 |
| 选项消息 | 附带按钮 | 给用户 2-4 个选项 |

## 六、交互细节

### 6.1 AI 主动填写

AI 自动填写的字段需要视觉区分：

| 填写来源 | 样式 |
|---------|------|
| AI 填写 | 绿色边框 + 浅绿背景 + "AI 已填写" badge |
| 用户手动修改 | 恢复默认边框 + "已编辑" badge |
| 用户通过对话修改 | 同 AI 填写，但消息中标注"已按你的要求更新" |

### 6.2 错误处理

| 场景 | 处理 |
|------|------|
| AI 生成失败 | 面板显示 "抱歉，生成失败了。你可以手动填写，或者换个方式描述需求" |
| AI 不可用（未配置 API Key） | 面板不出现，向导正常使用（降级为纯手动模式） |
| 语音识别失败 | 提示 "没有听清楚，可以打字告诉我吗？" |
| 网络错误 | 面板消息显示重试按钮 |

### 6.3 空状态

| 场景 | 处理 |
|------|------|
| 首页无配置 | 已有空状态设计（图标 + 引导文字） |
| 向导某步骤 AI 未生成 | 显示"等待上一步完成..." |
| 步骤未解锁 | dim 样式（半透明 + 灰度 + 不可交互） |

## 八、已明确的设计决策

### 8.1 AI 面板位置

- **默认**：跟随当前步骤卡片右侧，与步骤内容对齐
- **可拖拽**：用户可以拖动到任意位置
- **自动避让**：如果面板遮盖了"下一步"按钮或其他必须操作的控件，面板自动滑动避开
- **固定定位**：用户拖拽后，位置固定不变，不跟随滚动

### 8.2 面板最小化

- 最小化时缩成一个悬浮图标（🤖），停留在最后位置
- 双击图标恢复到完整面板
- 图标保持可拖拽

### 8.3 语音输入

- 所有输入框（首页、AI 面板）都放置 🎤 麦克风按钮
- **当前版本**：麦克风按钮仅作占位符（视觉完整，点击提示"语音输入即将支持"）
- **后续版本**：集成 Web Speech API 实现真实语音输入

### 8.4 对话历史

- 切换步骤时对话历史**保留**
- 刷新页面后对话历史**保留**（存入 localStorage 或后端）
- 每个步骤的消息带有步骤标记，方便回溯

### 8.5 多语言

- **当前版本**：仅支持中文
- **后续版本**：i18n 国际化，至少支持中文 + 英文

### 8.6 移动端适配

- **当前版本**：桌面端优先，移动端可基本使用（响应式布局）
- **后续版本**：移动端 AI 面板改为全屏底部抽屉（bottom sheet）替代浮动面板，触控优化
