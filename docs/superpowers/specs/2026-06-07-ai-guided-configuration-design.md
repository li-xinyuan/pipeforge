# ConfigForge AI 引导式配置 — 交互设计文档

> 版本: v1.0
> 日期: 2026-06-07
> 状态: 设计阶段，待确认后进入开发计划

---

## 一、设计目标

将 ConfigForge 从"手动填写表单 + AI 可选辅助"改造为"AI 手把手引导 + 用户审核确认"的协同配置模式。

**核心原则：**

1. **AI 主导引导，用户审核确认** — AI 主动提问、主动填写、主动推进流程，用户只需确认或微调
2. **对话与界面联动** — 用户在对话中的选择直接反映到表单，操作表单也会被 AI 感知并响应
3. **渐进式披露** — 步骤按序解锁，每步由 AI 引导完成后再进入下一步
4. **多模态输入** — 支持文字输入、选项按钮点击、语音输入三种方式，效果一致
5. **首页简洁** — 只有一个输入框，AI 面板在进入向导后才出现

**关键规则：**

- 首页：只有输入框，**不显示 AI 面板**

- 步骤按序解锁：当前步骤完成才解锁下一步
- 已完成的步骤可回看，但 AI 引导焦点始终在当前步骤

## 三、首页设计

### 3.1 布局：双入口设计

首页提供两条并行的配置创建路径——AI 引导为主，手动创建为备选。

```
┌──────────────────────────────────────────────┐
│  ⚡ ConfigForge                              │
├──────────────────────────────────────────────┤
│                                              │
│         ⚡ AI 驱动 · 你说需求，AI 填配置        │
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │ 💬 描述你的数据处理需求...          🎤 │    │  ← AI 可用时：高亮可交互
│  └──────────────────────────────────────┘    │  ← AI 不可用时：灰掉 + 提示"需配置 AI"
│              [✨ AI 引导配置]                  │  ← AI 可用时主入口
│                                              │
│         或 手动创建 →                         │  ← 始终可见，点击直接进入向导（无 AI）
│                                              │
│  试试这样说：                                  │  ← AI 可用时才显示
│  · 导出用户表到 CSV                           │
│  · 合并订单表和用户表，统计金额                  │
│                                              │
│  ──── 最近配置 ────                           │
│  ...                                         │
└──────────────────────────────────────────────┘
```

**AI 可用性检测**：复用现有 `AiStatusBanner` 组件中的 `useAiStatus` composable，在首页实时判断 AI 状态。

### 3.2 AI 可用时

| 元素 | 行为 |
|------|------|
| 输入框 | 高亮可输入，placeholder 显示示例 |
| 🎤 按钮 | 显示（当前版本为占位符，下一版本实现语音） |
| "AI 引导配置" 按钮 | 主入口，点击 → 进入向导 + AI 面板出现 |
| 示例 chips | 显示，点击自动填入并进入向导 |
| "手动创建 →" 链接 | 始终可见，备用入口 |
| Enter 键 | 等同于点击"AI 引导配置" |

### 3.3 AI 不可用时（降级策略）

| 元素 | 行为 |
|------|------|
| 输入框 | 灰掉 + 显示提示文案"需先配置 AI 才能使用智能引导" |
| 🎤 按钮 | 隐藏 |
| "AI 引导配置" 按钮 | 隐藏 |
| 示例 chips | 隐藏 |
| "手动创建 →" 链接 | **突出显示**，点击直接进入向导（无 AI 面板，纯手动模式） |
| "前往设置" 按钮 | 输入框内或旁边显示，引导用户配置 AI |

**降级规则**：
- AI 未配置或无 API Key → 首页自动显示降级状态
- 无需用户做任何判断，页面根据 AI 状态自动切换
- 手动创建的向导中不显示 AI 面板，其他功能不变

## 四、5 步向导 + AI 协同

### 4.1 进度指示器

```
[✓ 场景信息] → [⟳ 输入源] → [ 处理步骤] → [ 输出配置] → [ 导出执行]
   已完成       当前步骤      待解锁        待解锁        待解锁
```

- 已完成：绿色边框 + ✓ 图标
- 当前步骤：黄色边框 + 脉冲动画
- 待解锁：灰色，不可点击

### 4.2 步骤 1 — 场景信息

**AI 行为：**
- 进入时 AI 面板自动弹出第一条消息
- AI 从用户首页输入中提取场景名称，自动填入表单
- AI 生成场景描述，自动填入
- AI 提示用户确认或修改

**AI 面板消息示例：**
```
🤖 好的！我来帮你创建这个订单统计流水线。
   先看步骤 1——我已经帮你填好了场景名称，
   你可以直接修改。确认后我们进入步骤 2。

   [✅ 确认，下一步]
```

**表单状态：**
- 场景名称：已填写，绿色边框（AI 填充标记）
- 场景描述：已填写
- 版本号：默认 "1.0"

**注意**：AI 面板中的"确认"按钮只确认当前步骤内容，**不会自动导航到下一步**。用户需自己点击页面底部的"下一步"按钮推进。保持用户对流程的控制权。

### 4.3 步骤 2 — 输入源

**AI 行为：**
- AI 面板询问数据来源类型
- 同时，页面上的三个类型卡片可见
- 用户可以通过三种方式选择：
  1. 直接点击页面上的类型卡片
  2. 点击 AI 面板中的选项按钮
  3. 在 AI 面板输入框中打字（如 "Excel"）

**三种输入方式联动：**
```
用户点击卡片 "📊 Excel"
  → 卡片高亮，显示上传区域
  → AI 面板自动追加用户消息 "📊 Excel"
  → AI 面板回复 "好的，Excel 文件。请上传文件..."

用户在 AI 面板输入 "CSV"
  → AI 识别意图，面板显示选项确认
  → 页面上 CSV 卡片自动高亮
  → 上传区域切换到 CSV 模式

用户在 AI 面板点击按钮 "📊 Excel"
  → 同点击卡片的效果
```

**AI 面板消息示例：**
```
🤖 现在我们配置输入源。
   你的数据在哪种文件或系统里？
   可以直接点上方卡片，也可以在下面告诉我～

   [📊 Excel] [🗄 CSV] [🔌 数据库]

👤 📊 Excel

🤖 好的，Excel 文件。请上传文件，
   然后我们进入步骤 3 生成处理代码。

   [📤 我上传好了]
```

**表单状态：**
- 选择前：三个卡片等大，虚线边框
- 选择后：选中卡片实线绿色边框 + 浅绿背景，其余变暗
- 上传区域出现（虚线框 + 拖拽提示）

### 4.4 步骤 3 — 处理步骤

**AI 行为：**
- AI 根据场景描述 + 输入源信息，自动生成 SQL 或 Python 代码
- 代码显示在可编辑的代码块中
- AI 解释代码逻辑，标注假设（如关联字段名）
- 用户可编辑代码，或通过对话让 AI 修改

**AI 面板消息示例：**
```
🤖 我已经生成了 SQL 代码：
   用 JOIN 关联两个表，按城市分组统计订单数和金额。
   我假设关联字段是 user_id，不对的话在代码块里直接改～

   [✅ 确认，下一步] [✏ 我要修改]

👤 把统计改成按产品类型分组

🤖 好的，SQL 已更新为 GROUP BY product_type。
   Step 4 的列映射我也同步更新了。

   [✅ 确认]
```

**跨步骤联动（v1.0 简化方案）：**
- 用户从 Step 3 点"下一步"进入 Step 4 时，**前端直接比较 SQL SELECT 列名与 Step 4 已有列映射**
- 如果列名不一致，弹轻量提示条（不调 AI）：
  ```
  "SQL 输出列已变更（新增 2 列，移除 1 列），是否更新列映射？"
  [更新列映射]  [保持不变]
  ```
- 检测逻辑纯前端，只发生一次（步骤切换时）
- 用户编辑过程中不打断，不调用 AI

### 4.5 步骤 4 — 输出配置

**AI 行为：**
- AI 根据场景描述自动选择输出类型（Excel/CSV/数据库）
- AI 自动生成文件名模板
- AI 自动完成列映射（从 SQL SELECT 字段推断）
- 支持切换输出类型

**AI 面板消息示例：**
```
🤖 输出配置好了：Excel 格式，
   列映射已自动完成：city, order_count, total_amount
   我们进入最后一步吧！

   [✅ 确认] [🔄 换成 CSV]
```

**表单状态：**
- 输出格式：下拉选择（Excel / CSV / 数据库）
- 文件名：已填写，带模板变量（{{date}}）
- 列映射：表格显示，AI 已完成，可手动编辑
- 数据检查点：可选，AI 建议添加

### 4.6 步骤 5 — 导出执行

**AI 行为：**
- 确认全部配置完成
- 提供下载 YAML / 执行流水线两个操作

**AI 面板消息示例：**
```
🤖 🎉 全部配置完成！5 个步骤都就绪了。
   你可以下载 YAML 文件，或直接执行流水线。

   [🎉 太棒了]
```

## 五、AI 面板设计

### 5.1 布局：固定右侧面板

**v1.0 采用固定右侧布局**，复用现有 `AiChatPanel.vue` 的 sidebar 模式：

```
┌────────────────────────────────┬──────────────────────┐
│  步骤内容区（flex-1，可滚动）     │  🤖 Forge · AI 助手   │
│                                │                      │
│  Step 1: 场景信息               │  🤖 好的！我来帮你...   │
│  Step 2: 输入源                 │                      │
│  Step 3: 处理步骤               │       [✅ 确认]       │
│  Step 4: 输出配置               │                      │
│  Step 5: 导出执行               │  [输入框______] [发送] │
│                                │                      │
│              [下一步]            │                      │
└────────────────────────────────┴──────────────────────┘
```

- 面板固定宽度 360px，始终在右侧
- 步骤内容区占据剩余空间，可独立滚动
- **不做拖拽、不做跟随、不做自动避让**（v2.0 再考虑）
- 面板最小化时缩成右侧悬浮按钮，单击恢复（而非双击）

### 5.2 与现有组件的关系

**不新建 `AiGuidePanel.vue`，增强现有 `AiChatPanel.vue`**：

| 现有能力 | 引导模式需要 | 改动 |
|---------|-----------|------|
| sidebar / overlay / fullscreen 三模式 | 固定右侧面板 | 新增 `mode="guide"` |
| 聊天消息列表 | 引导消息 + 快捷操作按钮 | 增强消息类型，支持 `actions` 字段 |
| 发送文本消息 | 同 | 无改动 |
| loading 状态 + 打字机效果 | AI 思考等待提示 | 复用现有动画 +"正在思考..."文案 |
| 不感知步骤 | 需要知道当前步骤 | 新增 `currentStep` prop |
| 无对话历史持久化 | 刷新后保留 | 新增 localStorage 存取（最近 50 条） |

### 5.3 消息类型扩展

```typescript
interface ChatMessage {
  role: 'user' | 'ai'
  content: string
  step?: number              // 关联的步骤编号（1-5）
  type?: 'guide' | 'chat' | 'warning' | 'suggestion'
  actions?: GuideAction[]    // 快捷操作按钮
  prefill?: Record<string, any>  // AI 建议预填的数据
  timestamp: number
}

interface GuideAction {
  label: string              // 按钮文字
  value: string              // 点击后发送的消息
  style?: 'primary' | 'default' | 'warning'
}
```

### 5.4 "确认"按钮行为

用户点了确认按钮后，**AI 不自动导航**。只确认当前步骤，用户自己点"下一步"按钮推进。保持用户对流程的控制感。

### 5.5 AI 响应延迟处理

- **0-5 秒**：面板显示三点动画 + "AI 正在思考..."
- **5-30 秒**：追加提示 "生成较复杂的代码可能需要更长时间"
- **超过 30 秒**：显示"取消"按钮，用户点击后中断请求（AbortController），面板恢复为空输入状态，AI 消息保留"生成已取消"提示
- 复用现有 `AiChatPanel` 的 loading 状态和打字机效果

### 5.6 AI 生成有误时的回退

| 场景 | 处理 |
|------|------|
| SQL 语法错误 | 后端 dry_run 失败 → 面板显示错误详情 + "重新生成"按钮 |
| 用户说"不对" | AI 基于对话历史修正，保留上一条生成内容作为对比 |
| 部分正确 | 用户直接编辑代码 → 字段移除 AI 标记 → AI 不干预用户手动编辑 |

### 5.3 消息类型

| 类型 | 样式 | 用途 |
|------|------|------|
| 普通消息 | 深色气泡 | AI 的说明、提问 |
| 建议气泡 | 青色边框 | AI 的建议、提示 |
| 警告气泡 | 黄色边框 | 检测到关联影响，需用户确认 |
| 选项消息 | 附带按钮 | 给用户 2-4 个选项 |

## 六、交互细节

### 6.1 AI 主动填写

AI 自动填写的字段需要视觉区分：

| 填写来源 | 样式 |
|---------|------|
| AI 填写 | 绿色边框 + 浅绿背景 + "AI 已填写" badge |
| 用户手动修改 | 恢复默认边框 + "已编辑" badge |
| 用户通过对话修改 | 同 AI 填写，但消息中标注"已按你的要求更新" |

### 6.2 错误处理

| 场景 | 处理 |
|------|------|
| AI 生成失败 | 面板显示 "抱歉，生成失败了。你可以手动填写，或者换个方式描述需求" |
| AI 不可用（未配置 API Key） | 面板不出现，向导正常使用（降级为纯手动模式） |
| 语音识别失败 | 提示 "没有听清楚，可以打字告诉我吗？" |
| 网络错误 | 面板消息显示重试按钮 |

### 6.3 空状态

| 场景 | 处理 |
|------|------|
| 首页无配置 | 已有空状态设计（图标 + 引导文字） |
| 向导某步骤 AI 未生成 | 显示"等待上一步完成..." |
| 步骤未解锁 | dim 样式（半透明 + 灰度 + 不可交互） |

### 7.2 后端 API

**复用现有 `POST /api/ai/suggest`**，不新增独立端点。现有 API 已支持 `category: scene|columns|sql|mapping|chat`，拥有速率限制、错误清洗、超时处理等基础设施。

| 引导场景 | 实现方式 | 说明 |
|---------|---------|------|
| 首页 → 场景信息 | `suggest(category='scene', context=用户需求描述)` | 返回场景名称、描述、建议输入类型 |
| 步骤 2 输入类型询问 | `suggest(category='chat', context=当前状态)` | 返回引导消息 + 快捷操作按钮 |
| 步骤 3 代码生成 | `suggest(category='sql', context=输入源+场景)` | 已有能力，增强上下文 |
| 步骤 4 列映射 | `suggest(category='mapping', context=SQL输出列+目标列)` | 已有能力 |
| 步骤间引导消息 | `suggest(category='chat', context=当前步骤状态)` | 返回引导消息 + 预填数据 |

**新增 `GuideResponse` 类型**（前端）：

```typescript
interface GuideResponse {
  message: string              // AI 引导消息（显示在面板中）
  actions?: GuideAction[]      // 快捷操作按钮
  prefill?: Record<string, any>  // AI 建议预填的数据（字段路径 → 值）
}
```

前端新增 `useAiGuide` composable 封装引导逻辑，内部调用现有 `useAiApi` 的 `askSuggestion`。

### 7.3 Store 改动

- `wizard.ts`：`output` 默认值保持 `null`（已修改）
- 字段来源追踪简化：

```typescript
// wizard.ts — UI 层临时标记，不持久化，刷新后消失
const aiPrefilledFields = ref<Map<string, boolean>>(new Map())

function markAiPrefilled(fieldPath: string) {
  aiPrefilledFields.value.set(fieldPath, true)
}

function markUserEdited(fieldPath: string) {
  aiPrefilledFields.value.delete(fieldPath)
}
```

标记仅用于 UI 视觉区分（绿色边框 + "AI 已填写" badge 或 "已编辑" badge）。用户手动编辑某字段时移除标记。刷新页面后所有字段一视同仁。

## 八、已明确的设计决策

> 以下决策已根据设计评审反馈更新。

### 8.1 AI 面板位置（已修正）

- **v1.0**：固定右侧面板（360px），不跟随、不拖拽、不避让。复用 `AiChatPanel.vue` sidebar 模式
- **v2.0**：考虑可拖拽浮动面板 + 跟随步骤
- 面板最小化时缩成悬浮图标（🤖），**单击**恢复

### 8.2 语音输入（已修正）

- AI 可用时显示 🎤 按钮，不可用时隐藏——**不做占位符**
- 当前版本不集成语音（下一版本实现）
- 原因：可点击但不可用的按钮会让用户失望

### 8.3 对话历史

- 切换步骤和刷新页面后对话历史**保留**（localStorage）
- 注意 localStorage 5MB 限制：最近 50 条消息持久化，超过时自动截断旧消息
- 截断时面板底部显示提示"对话历史已截断，仅保留最近 50 条"

### 8.4 多语言

- **v1.0**：仅中文
- **后续版本**：i18n

### 8.5 移动端适配

- **v1.0 可直接复用**：现有 `AiChatPanel` 已有 `mode="fullscreen"` 全屏模式，移动端自动切换
- 桌面端用 `mode="guide"`（右侧面板），移动端用 `mode="fullscreen"`（全屏抽屉）
- **v2.0**：移动端进一步优化为原生 bottom sheet，触控手势支持

### 8.6 组件策略（新增）

- **不新建 `AiGuidePanel.vue`**，增强现有 `AiChatPanel.vue`
- 新增 `mode="guide"` 引导模式、`currentStep` prop、`actions` 消息类型
- 保留自由聊天模式作为备选（用户可随时切换）

### 8.7 API 策略（新增）

- **不新增独立 API 端点**，复用现有 `POST /api/ai/suggest`
- 前端新增 `useAiGuide` composable 封装引导逻辑
- 后端只需增强 `suggest` 的 prompt 模板

### 8.8 跨步骤联动（已修正）

- v1.0 不做 AI 驱动的实时联动检测
- 改为**步骤切换时前端比较列名**，不一致时弹提示条
- 纯前端逻辑，不调 AI，只在步骤切换时触发一次
