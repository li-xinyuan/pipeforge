## P0：硬阻塞修复

### P0-1：safe_identifier() 支持中文列名

**问题**：`src/pipeforge/core/sqlite.py` 第 9 行正则 `^[a-zA-Z_][a-zA-Z0-9_]{0,63}$` 拒绝中文列名，导致中国用户无法使用中文 Excel/CSV。

**修改文件**：`src/pipeforge/core/sqlite.py`

**修改内容**：
```python
# 旧
_IDENTIFIER_RE = re.compile(r'^[a-zA-Z_][a-zA-Z0-9_]{0,63}$')

# 新：支持 Unicode 字母（中文、日文等）
_IDENTIFIER_RE = re.compile(r'^[\w][\w]{0,63}$', re.UNICODE)
```

**注意**：SQLite 标识符用双引号引用时支持任意 Unicode 字符，所以正则放宽是安全的。但需确保 `safe_identifier()` 的引用逻辑（加双引号）不变。

**测试**：
- 运行 `pytest tests/ -k "safe_identifier or chinese"` 
- 手动测试：上传含中文列名的 Excel 文件

### P0-3：Python exec() 沙箱化

**问题**：`src/pipeforge/plugins/processor/python.py` 的 `exec()` 无限制，用户可执行任意系统命令。

**修改文件**：`src/pipeforge/plugins/processor/python.py`

**修改内容**：
1. AST 预检：解析代码 AST，白名单允许的节点类型（表达式、赋值、if/for/while、函数定义、return），禁止 import、exec、eval、__import__、open 等
2. `__builtins__` 白名单：只暴露 `print, len, range, int, float, str, list, dict, set, tuple, bool, None, True, False, min, max, sum, abs, round, enumerate, zip, map, filter, sorted, reversed, isinstance, type, hasattr, getattr`
3. 超时控制：`signal.alarm(30)` 限制执行时间

**测试**：
- 正常 Python 脚本（数据处理）应正常执行
- `import os; os.system('rm -rf /')` 应被拒绝
- `__import__('os').system('ls')` 应被拒绝
- 死循环 `while True: pass` 应超时
