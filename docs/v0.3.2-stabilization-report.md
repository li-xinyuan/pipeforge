## 修复清单

### P1: 修复 3 个异步测试失败

**问题**: `configforge/tests/test_api_files.py` 中的 3 个测试因 `@pytest.mark.asyncio` 未注册而失败，报错 `async def functions are not natively supported`。

**根因**: `pytest-asyncio` 未安装且 `pyproject.toml` 缺少 asyncio 配置。

**修复**:
- 安装 `pytest-asyncio>=1.4.0` 作为开发依赖
- 在 `pyproject.toml` 添加 `[tool.pytest.ini_options]` 配置：
  - `asyncio_mode = "auto"`
  - 注册 `asyncio` 标记

**涉及文件**: `pyproject.toml`

**验证**: `uv run pytest configforge/tests/test_api_files.py -v` → 3/3 通过

---

### P2: 添加 .vue 模块类型声明

**问题**: `npx tsc --noEmit` 报告 6 个 TS2307 错误（所有 `.vue` 导入无法解析）。虽然构建使用 `vue-tsc` 不受影响，但 IDE 和 CI 的 plain `tsc` 检查会报错。

**修复**: 新建 `configforge-web/src/env.d.ts`，声明 `*.vue` 模块类型。

```ts
/// <reference types="vite/client" />
declare module '*.vue' {
  import type { DefineComponent } from 'vue'
  const component: DefineComponent<{}, {}, any>
  export default component
}
```

**涉及文件**: `configforge-web/src/env.d.ts` (新建)

**验证**: `npx vue-tsc --noEmit` 和 `npx tsc --noEmit` → 0 错误

---

### P2: 修复 WizardStepCard 锁遮罩层 z-index

**问题**: `.wizard-step-card__lock-overlay` 缺少 `z-index`。当卡片内子元素（如 AiChatPanel 的 `z-index: 200`）建立自己的层叠上下文时，半透明锁遮罩可能被遮挡。

**修复**:
- 给 `.wizard-step-card` 添加 `z-index: 0`（配合 `position: relative` 建立层叠上下文）
- 给 `.wizard-step-card__lock-overlay` 添加 `z-index: 1`（确保遮罩位于卡片子元素之上）

**涉及文件**: `configforge-web/src/components/wizard/WizardStepCard.vue` (第 52 行、第 74 行)

**验证**: `WizardStepCard` 单元测试 6/6 通过

---

### P2: 更新 .gitignore

**问题**: 以下路径未被忽略，导致本地工具生成物和构建产物出现在 `git status` 中：
- `configforge/static/assets/` — Vue 前端构建输出
- `.claude/settings.local.json` — 本地 Claude Code 权限配置
- `.codegraph/` — CodeGraph 本地数据库
- `.understand-anything/` — Understand Anything 知识图谱

**修复**: 在 `.gitignore` 中添加 4 条规则。

**涉及文件**: `.gitignore`

**验证**: `git status` 确认 `.codegraph/` 和 `.understand-anything/` 不再显示为未跟踪

**注意**: `.claude/settings.local.json` 已被 git 追踪，`.gitignore` 规则不会影响已追踪文件。如需停止追踪，可执行 `git rm --cached .claude/settings.local.json`。
