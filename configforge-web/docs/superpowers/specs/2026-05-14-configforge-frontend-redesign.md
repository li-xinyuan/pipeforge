# ConfigForge 前端重设计 — 设计规格

## 背景

ConfigForge 当前前端为功能导向的 5 步向导界面，存在以下问题：
- 首页空洞（仅一段文字 + 一个按钮）
- 步骤切换生硬（页面跳转，无过渡动画）
- AI 反馈弱（仅按钮 spinner，成功/失败无区分）
- 卡片布局风格混乱（3 种不同选择器风格）
- 缺乏设计语言体系（阴影、间距、排版不一致）
- 无响应式设计
- 无暗色模式

## 目标用户

**业务人员 / 非技术用户**：需要 AI 辅助完成配置，不太熟悉 SQL 或技术概念。关注引导流程和智能提示。

## 设计方向

- **风格**：轻量活泼风 — 明亮色彩、圆角大、柔和阴影、emoji 图标、丰富微交互
- **主色**：青瓷绿 Teal `#0d9488` → `#14b8a6`（渐变）
- **主题**：浅色 + 暗色双主题

## 架构：单页滚动 + AI 三级嵌入

废弃当前的 5 个独立路由页面，改为单页纵向滚动布局，右侧可折叠 AI 面板。

### AI 三级嵌入策略

1. **卡片内联（一级）**：AI 分析结果直接展示在步骤卡片内 — 列类型识别、关联键检测、表名推荐。用户无需额外操作即可看到 AI 输出。
2. **建议条（二级）**：紫色渐变提示条出现在卡片内，提供上下文相关的 AI 建议 — "AI 已根据你的描述推荐了场景名称"。
3. **侧边面板（三级）**：右侧可折叠面板用于深度对话交互 — 多轮对话、快捷操作按钮（"AI 生成 SQL"、"自动添加输出列映射"）、用户可主动提问。

### 步骤卡片状态

| 状态 | 视觉 | 说明 |
|------|------|------|
| 已完成 | 白色卡片 + 细灰边框 + 绿色 `✓` 标签 | 用户已填写/确认 |
| 进行中 | 青瓷绿边框 + 微阴影 + `⟳` 标签 | 当前焦点步骤 |
| 待解锁 | 半透明(opacity:0.55) + 浅底色 | 前置步骤未完成，内容预览 |

### 进度条

Sticky 定位在滚动区域顶部，背景模糊。点击已完成/进行中的步骤可跳转。连线颜色表示进度：
- 已通过：薄荷绿实线
- 未到达：灰色线

### 每步独立色彩标识

每步卡片使用不同的图标底色微渐变，增强视觉辨识：
1. 场景信息：`#f0fdfa` 底 + `#0d9488` 图标
2. 数据源：`#ccfbf1` 底 + `#0f766e` 图标
3. 数据转换：`#fef3c7` 底 + `#d97706` 图标
4. 输出定义：`#fef2f2` 底 + `#dc2626` 图标
5. 导出：`#eff6ff` 底 + `#0284c7` 图标

## 组件风格

### 按钮

| 层级 | 样式 | 用途 |
|------|------|------|
| 主要操作 | 青瓷绿渐变 + 白色文字 + 阴影 `0 2px 8px rgba(13,148,136,0.25)` | CTA、下一步 |
| 次要操作 | 白色背景 + 青瓷绿边框 + 青瓷绿文字 | 取消、返回 |
| 文字按钮 | 无背景无边框，青瓷绿文字 | 轻量操作 |
| 危险操作 | `#dc2626` 背景 + 白色文字（扁平无渐变） | 删除 |

圆角统一 `10px`。

### 输入框

| 状态 | 样式 |
|------|------|
| 默认 | `1px solid #d1d5db`，border-radius:10px，padding:10px 14px |
| 聚焦 | `2px solid #5eead4` + `background:#fafdfb` |
| 错误 | `2px solid #fca5a5` + `background:#fef2f2` |

### 卡片

圆角统一 `18px`，内边距 `28px`。阴影仅在"进行中"状态使用：`0 4px 20px rgba(13,148,136,0.08)`。

### 标签

小圆角 `6px`，浅色背景 + 深色文字：
- AI 标签：`#f0fdfa` 底 / `#0f766e` 字
- 状态标签：按功能色
- 版本/类型标签：`#f8fafc` 底 / `#64748b` 字

### AI 元素

- **内联建议条**：`linear-gradient(135deg,#f0fdfa,#ccfbf1)` + `1px solid #99f6e4`，圆角 `12px`
- **AI 对话气泡**：同背景 + `border-radius: 14px 14px 14px 4px`
- **用户对话气泡**：`#f1f5f9` 底 + `border-radius: 14px 14px 4px 14px`

## 首页

### Hero 区域

- 渐变底 `linear-gradient(180deg,#fafdfb 0%,#fff 100%)`
- Badge：`AI 驱动的数据流水线配置工具`（青瓷绿底胶囊）
- 标题：`用自然语言描述需求 / AI 自动生成配置`（后半句渐变色）
- 副标题：功能说明
- CTA：主要按钮 `开始新配置` + 次要按钮 `查看使用指南`
- 快捷提示词：3 个胶囊式示例（"合并两个 Excel 表"、"CSV 数据过滤导出"、"多表关联汇总统计"），点击直接开始

### 配置列表

- 卡片式列表，每项显示：图标 + 名称 + 版本 + 数据源数 + 输出类型 + 相对时间
- 可执行配置显示绿色 `可执行` 标签
- 空状态：插画 + 引导文案

## 暗色模式

顶部导航常驻主题切换按钮 `☾`。切换时 CSS 变量全局生效，平滑过渡。Naive UI 的 `darkTheme` 与自定义 CSS 变量同步。

## 动画与过渡

| 场景 | 动画 |
|------|------|
| 步骤解锁 | 卡片从 `opacity:0.55` 过渡到 `1` + 边框绿色脉冲 |
| AI 分析完成 | 分析结果区域从 `max-height:0` 展开 + 淡入 |
| 进度条前进 | 连线从左到右绿色填充（`transition: width 0.4s ease`） |
| 卡片进入 | 新步骤卡片从下方 `translateY(20px)` 滑入 + 淡入 |
| 侧面板开关 | AI 面板 `width` 过渡 + `opacity` 淡入淡出 |

## 响应式

- **桌面端（≥1024px）**：左侧主内容 + 右侧 AI 面板（340px）
- **平板端（768-1023px）**：AI 面板默认折叠，点击展开浮层
- **手机端（<768px）**：单列布局，卡片 `border-radius` 缩小至 12px，进度条水平滚动，AI 面板全屏浮层

## 技术实现要点

1. **Naive UI 主题覆写**：通过 `NConfigProvider` 的 `themeOverrides` 覆盖默认样式，暗色模式使用 `darkTheme` + 自定义变量
2. **CSS 变量**：在 `style.css` 中定义 `--color-primary`、`--color-bg` 等变量，通过 `[data-theme="dark"]` 切换
3. **滚动驱动**：使用 Intersection Observer 检测当前可见步骤，更新进度条状态
4. **AI 面板**：右侧面板使用 `transition` 控制宽度，面板内容在折叠时 `v-if` 销毁以减少内存
5. **Steps 1-5 路由**：保留向后兼容，重定向至 `/config/new`
